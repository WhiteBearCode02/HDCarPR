# ECO-DRIVE INSIGHT 사업 계획서 및 상세 요구사항 (v1.0)

## 1. 프로젝트 개요
- **서비스명**: ECO-DRIVE INSIGHT (에코 드라이브 인사이트)
- **타겟 사용자**: 고유가 시대에 경제 운전을 원하는 현대자동차 차주, SDV 데이터 기반 코칭을 원하는 사용자
- **핵심 문제**: 기존 순정 앱은 결과 통계 중심이며, 연비 하락의 원인(인과관계)과 실행 가능한 개선 방법을 제공하지 못함
- **목표**: 현대 디벨로퍼스 API 기반 차량 데이터와 외부 환경 데이터를 결합해, AI가 구간별 연비 하락 원인을 추론하고 맞춤형 피드백을 제공

---

## 2. 시장·기술 분석 및 차별화

### 2.1 벤치마킹
- **마이현대**: 누적/결과 통계 위주, 인과 분석 기능 부족
- **테슬라 에너지 앱**: 소비 원인 분석 역량 우수하나 타 브랜드 호환성 및 내연기관 중심 분석 한계

### 2.2 차별화 전략
1. **데이터 융합 분석**
   - 차량 내부 데이터(CAN) + 외부 데이터(교통량, 경사도, 날씨) 통합
2. **설명 가능한 AI(XAI)**
   - SHAP 기반 기여도 제시 (예: 급가속 40%, 정체 30%, 공조장치 20%)
3. **구간별 정밀 진단**
   - 전체 평균이 아닌 1km 단위 또는 이벤트 기반 구간 진단

---

## 3. 시스템 아키텍처
- **Frontend**: React.js / Next.js
  - 대시보드, 연비 추이, 영향도 시각화 (Chart.js 또는 D3.js)
- **Backend (Main)**: Node.js + Express
  - 현대 API 연동, 비즈니스 로직, 인증/권한, CRUD API
- **Backend (AI)**: Python + FastAPI
  - XGBoost 추론, SHAP 분석, 진단 리포트 생성
- **Database**: PostgreSQL
  - 사용자·주행 이력·구간 분석 결과 저장
- **Infra**: Docker 컨테이너 기반 배포

> 설계 원칙: **관심사 분리(Separation of Concerns)** 및 서비스별 독립 스케일링

---

## 4. 기능 요구사항

### 4.1 인증/회원 (Auth)
- **F-101** 현대 통합 계정 연동(OAuth 2.0)
- **F-102** 자체 계정 추가 정보 관리(차종, 연식)
- **F-103** 비밀번호 단방향 해싱(bcrypt)

### 4.2 주행 데이터 관리 (Vehicle Data CRUD)
- **F-201 (Create)** 주행 종료 시 현대 API 데이터 수집·저장
- **F-202 (Read)** 과거 주행 기록 목록 조회
- **F-203 (Update)** 주행 메모 추가/수정
- **F-204 (Delete)** 주행 기록 삭제

### 4.3 AI 연비 분석 및 추론
- **F-301** 1km 단위 연비 하락 원인 추론
- **F-302** SHAP 기반 원인 기여도 수치화
- **F-303** 자연어 코칭 가이드 생성

### 4.4 관리자 기능
- **F-401** 가입자·데이터량 모니터링
- **F-402** 모델 성능 대시보드(RMSE 등) 시각화

## 6. 핵심 기술 및 알고리즘

### 모델 1: XGBoost + SHAP (필수)
- 정형 데이터 처리 효율 우수
- 피처별 기여도 해석 가능 (XAI)
- 운전자 행동 개선 유도에 적합

### 모델 2: Temporal Fusion Transformer (선택)
- 시계열 장기 의존성 반영
- 미래 연비 효율 예측 고도화 가능

### 데이터 엔지니어링
- 초 단위 CAN 데이터 수집
- 구간 집계(Windowing, Aggregation)로 분석 효율 최적화
- 외부 데이터와 시간·공간 기준 조인

## 7. 개발 로드맵 (4주)
1. **1주차**: 환경 구축(Docker, DB), 현대 API 인증
2. **2주차**: 주행 수집 파이프라인 + CRUD API
3. **3주차**: AI 모델 학습 + SHAP 분석 + FastAPI 연동
4. **4주차**: 프론트 시각화, 통합 테스트, 배포

## 8. 면접 대응 포인트 (요약)

### Q1. AI 서버를 분리한 이유?
- Node.js는 I/O 중심 요청 처리, Python은 수치연산·ML 생태계 강점
- MSA 분리로 독립 배포/스케일링 가능

### Q2. 연비 분석 핵심 로직?
- CAN + 외부 환경 피처 벡터 구성
- XGBoost 예측 대비 실제 연비 편차를 SHAP으로 역추적
- 정량(기여도) + 정성(코칭 문장) 동시 제공
